# Screenshots

Doc/README screenshots, stored as **WebP** (lossless, metadata stripped). Capture as PNG, then convert
with `cwebp -lossless` (see the conversion script in the repo history). Landscape; crop to the relevant
panel; the dark theme matches the GUI.

Currently used in the README:

| Filename | What it shows |
| --- | --- |
| `gui-live-data.webp` | GUI **Live Data** (Grouped) — the headline shot. |
| `gui-config.webp` | GUI **configuration** form (structured form + nav). |
| `gui-control.webp` | GUI **Control** tab — per-outlet actions + group actions. |
| `gui-paths.webp` | GUI **Paths** tab — generated MQTT/Prometheus/EmonCMS paths (cropped from a tall capture). |
| `home-assistant-outlet.webp` | An outlet device in Home Assistant — switch, sensors, delays, power-on. |
| `home-assistant-group.webp` | A OneView group device — rollup sensors, member switches, All On/Off/Reboot. |
| `home-assistant-history.webp` | A Home Assistant sensor history graph (power over time). |

Not yet captured (add the file here and a slot in the README if you want them): **`grafana.webp`** (a Grafana/Prometheus dashboard) and **`emoncms.webp`** (EmonCMS inputs/feeds).
